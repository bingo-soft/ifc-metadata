# Skill: Create and Update CHANGELOG

## Purpose

Создавать и обновлять `CHANGELOG.md` по правилам:
- `changelog_policy.md`
- `version_policy.md`

## Command triggers

- `обнови changelog`
- `создай changelog`
- `завершить`
- `завершить и мерж`

## Preconditions

1. Должен существовать `changelog_policy.md`.
2. Должен существовать `version_policy.md`.
3. Должна быть известна целевая версия `X.Y.Z.T`.

Если любой precondition не выполнен:
- статус: `NOT READY`
- вернуть причину и список отсутствующих входных данных/файлов.

## Output format

Структура версии в `CHANGELOG.md`:

1. Заголовок версии:
   - `## [X.Y.Z.T] - YYYY-MM-DD HH:mm`
2. Обязательная секция:
   - `### Summary`
   - 1–3 пункта с общим смыслом изменений.
3. Затем секции Keep a Changelog по необходимости:
   - `### Added`
   - `### Changed`
   - `### Fixed`
   - `### Removed`
   - `### Security`

Секция `Unreleased` не используется.

## Update algorithm

1. Прочитать `CHANGELOG.md`.
2. Проверить, есть ли заголовок `# Changelog`:
   - если нет — создать файл с этим заголовком.
3. Сформировать новый блок версии в формате из `changelog_policy.md`.
4. Вставить блок новой версии **сразу после `# Changelog`** (лог растёт вверх).
5. Заполнить `### Summary` обязательным общим смыслом изменений.
6. Разнести изменения по секциям Keep a Changelog (`Added/Changed/Fixed/...`).
7. Отфильтровать шумовые пункты:
   - не добавлять банальные записи вроде «обновили changelog».
8. Проверить, что дата и время присутствуют в заголовке версии.

## Quality rules

- Писать только значимые изменения для пользователей/разработчиков.
- Не дублировать одни и те же пункты в нескольких секциях.
- Не добавлять пустые секции.
- Текст должен быть коротким и предметным.

## Source of truth

- Содержание/структура changelog: `changelog_policy.md`
- Расчёт версии: `version_policy.md`
