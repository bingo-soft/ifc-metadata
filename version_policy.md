# Version Policy

> Единый источник правил версионирования для всех skill/policy файлов репозитория.

## Format

Формат версии: `X.Y.Z`.

## Version string rule

- Префикс `v` в номере версии не используется (нельзя: `v2.15.10.47`, правильно: `2.15.10.47`).

## Source fields

- Базовый источник версии проекта: `AssemblyVersion` (если в проекте принят такой источник).
- При наличии `Version`/`FileVersion` они должны быть синхронизированы с `AssemblyVersion` по правилам репозитория.

## Usage rule

- `AGENTS.md`, `.codex/skills/*` не дублируют детальные правила версионирования.
- Эти файлы должны ссылаться на `version_policy.md` как на единый источник истины.
